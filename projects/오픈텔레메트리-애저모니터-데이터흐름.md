# OpenTelemetry to Azure Monitor Flow

이 문서는 OpenTelemetry를 활용하여 애플리케이션 로그, 트레이스, 메트릭 데이터가 Azure Monitor로 전달되는 과정의 전체 흐름을 보여줍니다. `fastapi_app.py` 샘플 코드를 기준으로 작성되었습니다.

## 아키텍처 다이어그램

```text
[앱 코드]
   │
   ├── logger.info("...")           ← Python 표준 logging
   ├── tracer.start_as_current_span() ← OpenTelemetry Trace API
   │
   ▼
[OpenTelemetry SDK 파이프라인]
   │
   ├── TracerProvider  ─── SpanProcessor(들)
   │       └── DropUnknownRouteProcessor (커스텀 SpanProcessor, 미식별 라우트 제거)
   │
   ├── LoggingHandler  ─── GlobalDimensionsFilter (커스텀 필터)
   │       └── custom_dimensions에 5W1H 비즈니스 컨텍스트 자동 주입
   │
   ▼
[Azure Monitor Exporter]  ← configure_azure_monitor() 실행 시 자동 구성
   │
   ├── Traces  → AppRequests / AppDependencies / AppTraces (Span 데이터)
   ├── Logs    → AppTraces (logger.xxx 로깅 데이터)
   └── Errors  → AppExceptions (예외 데이터)
   │
   ▼
[Azure Application Insights]
   └── Log Analytics Workspace → KQL 스크립트를 통한 데이터 조회 (union AppRequests, AppTraces 등)
```

## 핵심 개념 요약

1. **계측 계층 (Instrumentation)**
   - `FastAPIInstrumentor`: FastAPI 프레임워크에 붙어 HTTP 요청/응답에 대한 Span을 자동 생성합니다. (Trace ID 생성 시작점)
   - `logger`: 일반 파이썬 `logging` 모듈을 사용하되 OTel 핸들러를 붙여 표준화된 포맷으로 가공합니다.
2. **OpenTelemetry SDK 계층**
   - 개발자가 호출하는 "Trace API" (예: `tracer.start_as_current_span()`)의 실제 구현체입니다.
   - 데이터가 외부로 나가기 전(Exporter 직전)에 커스텀 속성을 끼워 넣거나 필터링(예: `DropUnknownRouteProcessor`, `GlobalDimensionsFilter`)할 수 있는 훅(Processor, Handler)을 제공합니다.
   - **Batch Processing (배치 전송):** `BatchSpanProcessor` 및 `BatchLogRecordProcessor`를 통해 데이터를 VM 메모리(Queue 버퍼)에 임시 저장합니다. 백그라운드 스레드가 동작하며, 일정 시간 주기(예: 5초)나 큐의 최대 전송 크기(예: 512개)에 도달할 때 버퍼에서 데이터를 꺼내어 Azure로 일괄 전송(HTTP POST)합니다. 이를 통해 애플리케이션 성능 저하를 방지합니다.
3. **Exporter 계층 (`azure.monitor.opentelemetry`)**
   - OTel 표준 데이터를 Azure Application Insights 전용 포맷(Envelope JSON)으로 직렬화(변환)하여 Ingestion Endpoint로 HTTP 요청을 보냅니다.
   - 앱이 클라우드 종속성(Vendor Lock-in)에서 벗어나게 해주는 핵심 컴포넌트입니다.
   - **Offline Storage (오프라인 버퍼링):** 네트워크 단절 등으로 전송이 실패할 경우, 해당 배치 데이터를 임시 경로(Windows: `%TEMP%`, Linux: `/var/tmp/`)의 디스크 파일로 저장 보관했다가 연결 복구 시 재전송하여 데이터 유실을 방지합니다.
